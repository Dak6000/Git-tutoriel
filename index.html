Chapitre 4

Commandes Essentielles

Les commandes Git que vous utiliserez quotidiennement

git init

Initialise un nouveau dépôt Git dans le dossier courant. 

git init
Démarrage
git add

Ajoute des fichiers à l'index (staging area). 

git add .
Staging
git commit

Crée un commit avec les fichiers de l'index. 

git commit -m "Message"
Sauvegarde
git status

Affiche l'état des fichiers (modifiés, indexés, etc.). 

git status
Information
git branch

Gère les branches : création, liste, suppression. 

git branch nouvelle
Gestion
git checkout

Change de branche ou restaure des fichiers. 

git checkout main
Navigation
git merge

Fusionne une branche dans la branche courante. 

git merge feature
Fusion
git clone

Télécharge un projet et son historique. 

git clone [url]
Téléchargement
git pull

Récupère les modifications du dépôt distant. 

git pull origin main
Réception
git push

Envoie les commits locaux vers le distant. 

git push origin main
Envoi
git remote -v

Affiche les URLs des dépôts distants. 

git remote -v
Configuration
git log

Affiche l'historique des commits. 

git log --oneline
Historique
git diff

Affiche les différences entre index et travail. 

git diff
Inspection
git reset

Retire un fichier de l'index. 

git reset file
Annulation

Comment collaborer efficacement sur des projets open source ou en équipe

Chapitre 6

Bonnes Pratiques

Les règles d'or pour garder un historique propre et lisible

best_practices.md
## 1. Commit Messages

Soyez clair et concis. Utilisez l'impératif. 

- Fix bug
+ Fix login page validation error on submit
## 2. .gitignore est votre ami

Ne committez jamais les dossiers node_modules, les fichiers de configuration locaux (.env) ou les fichiers compilés.

## 3. Atomic Commits

Un commit doit faire une seule chose, et la faire bien. Ne mélangez pas "Correction CSS" et "Refonte de la base de données".

## 4. Pull Before Push

Toujours récupérer les dernières modifications distantes avant d'envoyer les vôtres pour éviter les conflits. 

git pull origin main --rebase
